		- Gemini 3 Flash thinking take on bitwise logic:
			- Think of bitwise logic as a set of four "light switches" inside every cell. Each switch controls one wall. When you flip a switch "ON" (1), the wall exists. When it’s "OFF" (0), the path is open.
			  
			  Since you have 4 directions, you have a 4-bit number. The maximum value is <!---->1111 in binary, which is **15** in decimal or **F** in hexadecimal.<!---->

			- ### 1. The Mapping (Bitmasking)
			  
			  To understand this, you need to look at **Bitmasking**. This is the standard way programmers store multiple True/False values in a single number.
			- **North (N):** 20=1 (Binary: `0001`)
			- **East (E):** 21=2 (Binary: `0010`)
			- **South (S):** 22=4 (Binary: `0100`)
			- **West (W):** 23=8 (Binary: `1000`)
			  
			  **How to calculate a cell value:**
			  If a cell has a wall on the **North** and **South**, you add their values: 1+4=5.
			- In binary: `0101`
			- In hex: `5`
			  
			  If a cell is completely enclosed (the "42" pattern cells): 1+2+4+8=15.
			- In binary: `1111`
			- In hex: `F`

			- ### 3. Practical Implementation Tips
			- #### Setting a wall (Bitwise OR)
			  
			  When your generator decides a wall should exist, you "OR" the bit into the cell's current value:
			  
			  ```
			  # Start with an empty cell (0)
			  cell_value = 0 
			  
			  # Add a North wall (1)
			  cell_value |= 1 
			  
			  # Add a West wall (8)
			  cell_value |= 8
			  
			  # cell_value is now 9 (Binary 1001)
			  ```
			- The "Coherent" Data Rule
			  
			  Your project instructions say: *"each neighbouring cell must have the same wall if any."*
			  This is the trickiest part of bitwise logic in mazes. If you set an **East** wall for cell (0,0), you **must** also set a **West** wall for cell (1,0).

			- ### 4. A Teacher's Note on Debugging
			  
			  When you start coding, write a small helper function called `print_binary(value)`. It is much easier to debug `0101` than it is to debug `5` or `A`. Once the binary logic is perfect, the Hex conversion will happen automatically.
			- To make your life easier, copy this table into your notes. This is the "Key" to your bitwise logic:
			- | **Direction** | **Bit** | **Value (Decimal)** |
			  | ---- | ---- | ---- |
			  | **North** | $2^0$ | 1 |
			  | **East** | $2^1$ | 2 |
			  | **South** | $2^2$ | 4 |
			  | **West** | $2^3$ | 8 |
